# API 接口列表

## 📋 接口总览

本系统共包含 7 个控制器，30+ 个 API 接口，涵盖用户认证、分享帖管理、预约管理、取餐码管理、积分管理、统计分析和健康检查等功能。

| 模块 | 控制器 | 接口数量 | 基础路径 | 认证要求 |
|------|--------|---------|---------|---------|
| 用户认证 | AuthController | 4 | `/api/auth` | 部分公开 |
| 分享帖管理 | SharePostsController | 6 | `/api/shareposts` | 部分公开 |
| 预约管理 | ReservationsController | 6 | `/api/reservations` | 需要登录 |
| 取餐码管理 | PickupCodesController | 4 | `/api/pickupcodes` | 需要登录 |
| 积分管理 | KarmaPointsController | 6 | `/api/karmapoints` | 需要登录 |
| 统计分析 | StatsController | 2 | `/api/stats` | 需要登录 |
| 健康检查 | HealthController | 1 | `/health` | 公开 |

---

## 🔐 一、用户认证模块

### 1. 用户注册

**接口**：`POST /api/auth/register`

**权限**：公开

**描述**：创建新用户账号，密码使用 BCrypt 加密存储。

**请求体**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名，唯一 |
| email | string | 是 | 邮箱，唯一 |
| password | string | 是 | 密码 |
| confirmPassword | string | 是 | 确认密码 |

**响应**：
| 字段 | 类型 | 说明 |
|------|------|------|
| user | object | 用户信息 |
| token | string | JWT 访问令牌 |

---

### 2. 用户登录

**接口**：`POST /api/auth/login`

**权限**：公开

**描述**：验证用户凭据，生成 JWT Token 用于后续认证。

**请求体**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| usernameOrEmail | string | 是 | 用户名或邮箱 |
| password | string | 是 | 密码 |

**响应**：
| 字段 | 类型 | 说明 |
|------|------|------|
| user | object | 用户信息 |
| token | string | JWT 访问令牌 |

---

### 3. 获取当前用户信息

**接口**：`GET /api/auth/me`

**权限**：需要登录

**描述**：获取当前登录用户的详细信息。

**响应**：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 用户ID |
| username | string | 用户名 |
| email | string | 邮箱 |
| avatar | string | 头像URL |
| totalKarmaPoints | int | 总积分 |
| createdAt | datetime | 创建时间 |

---

### 4. 更新个人资料

**接口**：`PUT /api/auth/me`

**权限**：需要登录

**描述**：更新当前登录用户的个人信息。

**请求体**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| email | string | 是 | 邮箱 |
| avatar | string | 否 | 头像URL |

**响应**：
| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 用户ID |
| username | string | 用户名 |
| email | string | 邮箱 |
| avatar | string | 头像URL |
| totalKarmaPoints | int | 总积分 |

## 📝 二、分享帖管理模块

### 1. 获取分享帖列表

**接口**：`GET /api/shareposts`

**权限**：公开

**描述**：分页获取分享帖列表，支持关键词搜索。

**查询参数**：
| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| pageNumber | int | 否 | 1 | 页码 |
| pageSize | int | 否 | 10 | 每页条数 |
| searchTerm | string | 否 | - | 搜索关键词 |

**响应**：
| 字段 | 类型 | 说明 |
|------|------|------|
| items | array | 分享帖列表 |
| totalCount | int | 总条数 |
| pageNumber | int | 当前页码 |
| pageSize | int | 每页条数 |
| totalPages | int | 总页数 |

| foodType | string | 食物类型 |

| longitude | double | 经度 |

## 🔄 通用响应格式

所有接口均采用统一的响应格式：

```json
{
  "success": true,
  "message": "操作成功",
  "data": { ... }
}
```

### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| success | bool | 请求是否成功 |
| message | string | 响应消息 |
| data | object/array | 响应数据 |

### 错误响应示例

```json
{
  "success": false,
  "message": "用户名已存在",
  "data": null
}
```

## 🔐 认证说明

### Token 使用方式

需要认证的接口必须在请求头中携带 JWT Token：

```
Authorization: Bearer <your-token-here>
```

### Token 有效期

- Token 有效期为 24 小时
- 过期后需要重新登录获取新 Token

### 认证失败响应

```json
{
  "success": false,
  "message": "用户未认证",
  "data": null
}
```

## 📝 状态码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 201 | 创建成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |
| 503 | 服务不可用 |

---

## 💡 注意事项

1. 所有时间字段均使用 ISO 8601 格式（UTC 时间）
2. 请求和响应均使用 UTF-8 编码
3. 分页接口默认每页 10 条数据，最大不超过 100 条
4. 搜索关键词不区分大小写
5. 所有需要用户认证的接口请确保 Token 有效
6. 涉及权限的操作会严格校验用户身份
